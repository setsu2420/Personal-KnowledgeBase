# 智能情报分析平台 - 系统设计文档

## 一、系统概述

本系统是一个面向军事/防务情报分析场景的智能知识管理平台，基于 RAG（检索增强生成）架构与知识图谱技术，帮助情报分析人员高效管理、检索和分析各类情报资料。系统支持多格式资料上传（Word、PDF、TXT、OFD、JPG等）、语义检索、智能问答、知识图谱可视化、跨资料结论冲突检测及分析报告自动生成等功能。所有数据和模型推理均在本地私有化部署，保障信息安全。

系统围绕四大核心功能构建：**资料获取**、**分析报告**、**智能问答**、**知识图谱**。

**技术栈：** 前端 Vue 3 + Element Plus + TypeScript + 后端 Spring Boot 4.1 + MyBatis-Plus + FAISS/Vector 向量索引 + 外部大模型服务（DeepSeek-V3 / SiliconFlow API）

**部署方式：** 服务器私有化部署 + 开发调试环境

## 三、前台页面（用户侧）

前台提供4个核心功能卡片入口：

### 3.1 资料获取

**功能定位：** 情报资料的上传、入库与管理入口。

**核心功能：**
- **多格式支持：** 支持 Word (.docx)、PDF、TXT、OFD、JPG 等格式上传
- **两级分类匹配：** 上传时按一级/二级标签体系自动匹配分类
- **拖拽上传 & 文件夹导入：** 支持拖拽文件和递归导入文件夹，保留目录结构
- **元信息提取：** 自动识别标题、编写单位、编写日期、关键词、密级
- **语义分块预览：** 展示语义分块结果，每个块的 Token 数和内容预览
- **摄入队列可视化：** 显示进度条、排队/处理中/失败任务，支持取消和重试
- **SHA256 增量缓存：** 上传前检查文件哈希，未变更则跳过，节省处理时间
- **MinerU PDF 解析：** 可选的 PDF 高级解析（表格、公式、密集排版）

**处理流程：**
1. 用户选择/拖拽资料文件
2. 填写一级/二级分类标签
3. SHA256 检查 → 新文件进入摄入队列
4. 格式解析 → 元信息提取 → 语义分块
5. Embedding 向量化 → 写入向量数据库

### 3.2 分析报告

**功能定位：** 基于多份资料自动生成分析报告，支持多种分析类型。

**核心功能：**
- **主题选择：** 按一级标签选择分析主题领域
- **分析类型：** 技术对比分析、趋势研判、风险评估、综合态势分析
- **结构化输出：** 生成包含摘要、核心发现、对比分析、趋势判断的报告
- **引用来源列表：** 分析中引用的所有资料及章节位置
- **跨资料冲突检测：** 自动识别多份资料间的结论矛盾或数据不一致

### 3.3 智能问答

**功能定位：** 基于知识库的智能问答，给出带原文引用的精准答案。

**核心功能：**
- **自然语言提问：** 支持各类情报分析问题
- **带引用回答：** 答案标注来源资料和章节，逐句可溯源
- **置信度评分：** 基于检索结果数量、语义相似度、来源权威性的综合评分
- **推荐问题：** 预设高质量问题，覆盖常见分析方向
- **对话历史：** 保留历史问答记录
- **向量语义搜索：** 基于 FAISS 的高性能 embedding 检索

**技术实现（RAG）：**
1. 用户提问 → BGE-M3 Embedding 向量化
2. FAISS 向量检索 Top-K 相关语义块
3. Rerank 重排序提升精度
4. 知识图谱检索相关实体及关系
5. 拼接上下文 → 大模型生成带引用答案
6. 后处理：引用校验、置信度计算

### 3.4 知识图谱

**功能定位：** 知识关联可视化与深度分析。

**核心功能：**
- **四信号关联度模型：**

| 信号 | 权重 | 描述 |
|------|------|------|
| 直接链接 | x3.0 | 通过引用/链接直接关联的资料 |
| 来源重叠 | x4.0 | 共享同一原始来源的资料 |
| Adamic-Adar | x1.5 | 共享共同邻居的资料 |
| 类型亲和 | x1.0 | 相同分类标签的加分 |

- **Louvain 社区检测：** 自动发现知识聚类，内聚度评分
- **图谱可视化：** 力导向图布局，按分类或社区着色，按关联度调整边粗细
- **图谱洞察：** 惊奇连接检测（跨社区、跨类型的意外关联）、知识空白发现（孤立节点、稀疏社区）
- **交互操作：** 拖拽、缩放、悬停高亮邻居、点击跳转详情

## 六、核心数据流

```

→ 一级/二级分类匹配 → 语义分块 → Embedding向量化 → 写入向量库
```

```
用户提问 → Embedding向量化 → 向量检索Top-K + 图谱检索 
→ 拼接上下文 → 大模型生成 → 引用校验 → 返回结果
```

**知识图谱构建流程：**
```
资料集合 → 实体抽取 → 关系识别 → 四信号关联度计算 
→ Louvain社区检测 → 图谱可视化 → 洞察生成
```

## 九、多项目隔离与开发者 API 接入

### 9.1 项目隔离机制
系统支持多项目并行管理。为了确保各项目间数据绝不互通，系统实施了严密的隔离策略：
1. **数据库隔离**：知识条目、原始文档、知识图谱等表结构均包含 `project_id` 属性。
2. **Header 请求头拦截**：前端向后端发起的所有 API 请求均会在 Header 中携带 `X-Project-Id: <ID>`，由 Spring Boot 的 Interceptor 拦截后存入 ThreadLocal 的 `ProjectContext` 隔离上下文，从而在 MyBatis-Plus SQL 执行时动态拼装隔离条件。
3. **路由即时刷新**：在项目切换时，前台与后台主布局的容器视图通过绑定 `:key="currentProjectId"` 动态重载路由组件，确保数据与视图刷新一致。

### 9.2 多模态与图表展示重构
- **图表展示只留原始图**：图表资料库中移除了文件名和文字信息的噪音遮挡，直接平铺展示图表的原始图像，并支持横向顺畅左右滑动查看，点击直接调用图片媒体服务全尺寸放大。
- **长文档资料库列表化**：由于 PDF、Word 和 TXT 文件缺乏视觉封面，为避免卡片式占位符冗余，统一采用与词条百科类似的单行列表呈现，并根据扩展名渲染各色 Badge 标签（红色 PDF、蓝色 Word、绿色 Excel/CSV、橙色 PPT 等）。
- **外网链接自动爬取与跳转**：当通过 URL 爬取网络正文存入动态信息库后，系统识别为 `LINK` 类型（紫色 Badge），点击此文件可以直接以新窗口跳转源网页访问。
